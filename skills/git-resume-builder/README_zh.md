# Git 简历生成器

从 Git 提交记录中提取并结构化项目经验，生成可直接用于简历的内容。

## 描述

自动分析 Git 提交记录，生成量化、聚焦价值、符合简历规范的项目描述。遵循核心原则：成果量化、技术具体化、突出价值、简洁有力。

## 触发词

当用户提到以下内容时使用本技能：
- "整理项目经验"
- "生成简历项目"
- "git 提交 简历"
- "项目经验整理"
- "resume from git"
- "project experience from commits"

## 核心原则

1. **成果量化** - 使用具体数字（百分比、数量、时间节省、性能提升）
2. **技术具体化** - 列出具体技术栈、框架、工具
3. **突出价值** - 聚焦业务影响和技术成就
4. **简洁有力** - 符合简历格式，要点式呈现

## 工作流程

1. **收集 Git 数据**：提取目标作者的提交历史、统计信息和文件变更
2. **分析项目结构**：识别技术栈、模块和贡献模式
3. **提取关键信息**：项目周期、代码影响、技术栈、模块归属、核心功能
4. **生成简历内容**：结构化输出，包含项目描述、技术栈、角色和量化成果

## 使用示例

```
帮我用 git 提交记录整理项目经验，作者：tanpenggood
```

技能将：
- 分析作者 "tanpenggood" 的 Git 提交记录
- 计算项目周期、提交次数、代码变更量
- 从 package.json 和文件扩展名识别技术栈
- 生成可直接用于简历的项目经验部分

## 输出格式

每个项目遵循以下结构：
```markdown
### [项目名称]
**时间**: YYYY.MM - YYYY.MM  
**角色**: [前端工程师 / 后端工程师 / 全栈工程师 / 技术负责人]

**项目描述**:  
[项目简介、服务对象、核心目标，1-2 句话]

**技术栈**:  
[前端]: [具体框架、库]  
[后端]: [具体框架、语言]  
[基础设施]: [数据库、云服务、CI/CD]

**主要职责与成果**:
• [量化成果 1 - 使用百分比、数量、时间、性能等指标]
• [量化成果 2 - 聚焦交付价值]
• [量化成果 3 - 解决的技术复杂度]
```

## 质量检查清单

输出前验证：
- [ ] 每个要点至少包含一个数字/指标
- [ ] 技术栈列出具体版本或库名称
- [ ] 业务价值明确（不仅是"优化"而是"将 X 降低 Y%"）
- [ ] 总长度符合简历格式（3-5 个项目，每个 4-6 行）
- [ ] 使用动作动词（实现、优化、构建、重构）

## 要求

- 包含提交记录的 Git 仓库
- 目标作者的 Git 配置名称或邮箱
- Node.js 环境（用于读取 package.json）

## 注意事项

- 将 `<author>` 替换为用户的 Git 配置名称或邮箱
- 对于单体仓库，分别分析每个包/子项目
- 当提交信息模糊时（如"修复 bug"、"更新代码"），通过文件路径和差异统计推断
- 始终与用户确认：作者名称/邮箱、需包含的项目、简历格式（中文/英文）
